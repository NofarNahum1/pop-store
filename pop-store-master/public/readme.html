(1) Store Name: Pop-Up Store

(2) We sell Funko POP! figures - vinyl figurines of characters from pop culture, movies, and real life.

(3) The additional pages we added are:

  • pop-of-the-month.html:

    This page shows the Pop of the Month chosen by the admin along with a description.
    The admin can update the Pop of the Month and its description in the pop-of-the-month section of admin-dashboard.html, choosing only from Pops that are already in the store's inventory.


  • mystery-pop.html:

    This page gives clients an opportunity to enter a raffle for a mystery Pop.
    The user has three chances to guess correctly. when guessed correctly the user needs to fill his email and address

  • reviews.html and admin-reviews.html:

    This page allows clients to leave a review about their experience in our store.
    Simply write your review and click the 'Send' button on the reviews.html page.
    The admin can view all the submitted reviews on admin-reviews.html.

  • statistics.html:

    This page displays the store's sales statistics in a chart and a table for admins only.

(5) Specify all the different routes your app supports

  • '/readme.html'

    This route serves the readme.html file tp the client.

  • /api/best-sellers

    This route serves part of the purchases data.

  • '/api/submit-guess'

    This route handles the submission of guesses from users.

  • '/api/log'

    This route handels the log in requests.

  • '/api/logout'

    This route handels the log out requests.

  • '/api/admin/logout'

    This route handels the admin's log out requests.

  • '/register'

    This route serves the register.html file to the user.

  • '/login'

    This route serves the login.html file to the user.

  • '/store'

    This route serves the store.html file to the user.

  • '/cart'

    This route serves the cart.html file to the user.

  • '/statistics'

    This route serves the statistics.html file to the user.

  • '/api/products'

    This route handles GET requests to fetch all products.

  • '/api/products/search'

    This route handles GET requests to search for products based on a query.

  • '/api/cart/items'

    This route handles GET requests to fetch items in the user's cart.

  • '/api/cart'

    This route handles POST requests to add a product to the user's cart.

  • '/api/cart/delete'

    This route handles DELETE requests to remove a product from the user's cart.

  • '/api/cart/clear'

    This route handles POST requests to clear the user's cart.

  • '/api/purchase'

    This route handles purchase-related requests.

  • '/api/admin/logs'

    This route handles GET requests to fetch admin logs.

  • '/api/admin/purchases'

    This route handles GET requests to fetch all purchases.

  • '/api/admin/pop-of-the-month'

    This route handles POST requests to set the Pop of the Month.

  • '/api/pop-of-the-month'

    This route handles GET requests to fetch the Pop of the Month.

  • '/api/reviews'

    This route handles POST requests to submit user reviews.

  • '/api/admin/reviews'

    This route handles GET requests to fetch all reviews for the admin.

  • '/admin/products'

    This route handles POST requests to add a new product (with image upload).

  • '/admin/products'

    This route handles DELETE requests to remove a product.
